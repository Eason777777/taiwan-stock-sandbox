# 錢錢錢市 股票模擬系統 規則 V1

## 專案概述

本系統旨在提供一個高度擬真的台股歷史交易模擬環境。玩家可自由選擇歷史時間起點，在「平行宇宙」中練習投資策略。系統核心強調「時空隔離」與「邏輯防呆」，確保玩家無法利用未來資訊進行套利，並在簡化的 MVP 模型下維持金融運作的嚴謹性。

---

## 此版本不考慮

- 除權息
- 減資
- T+2 交割制度
- 融資融券
- 其他複雜金融工具（如選擇權、期貨等）
- 圈存交易機制
- IOC (Immediate or Cancel) 與 FOK (Fill or Kill) 等特殊委託類型
- 零股交易
- 部分成交

## 一、登入

### 1. 使用者註冊

有註冊功能，使用者需要輸入 帳號( account ) 和 密碼 ( password )，系統檢查，就註冊成功。

### 2. 登入流程
有個登入介面

玩家輸入帳號密碼後，系統驗證其有效性，並引導玩家進入主選單以選擇或建立新的存檔。


### 3. 單一登入原則

同一帳號同一時間僅允許一組有效登入憑證。若同一帳號再次登入，系統將覆蓋前一次登入憑證。避免異地登入以及異地操作

## 二、模擬存檔管理

### 1. 建立新模擬存檔 (Create Save)

- 玩家需輸入「存檔名稱」、選擇「初始歷史日期」及「起始資金」。
- 起點日期不得早於系統資料庫最早紀錄。
- 起點日期不得晚於系統資料庫最後可用交易日。
- 使用者不得選擇「沒有交易日」的日期作為起點，系統應提示並要求重新選擇。
- 使用者可選擇由系統隨機抽取一個有效歷史交易日作為起點。
- 使用者可選擇由系統隨機決定初始資金，範圍為 <font color="red">50,000 </font> 元至 <font color="red">1,000,000 </font> 元。
- 初始資金於建檔時全數配發至「存款戶」。

### 2. 建檔限制
使用者可選擇建立存檔
- 每位玩家最多可建立 <font color="red">5 </font> 個「進行中」存檔。
- 每位玩家最多可保留 <font color="red">10 </font> 個存檔，包含進行中、已破產與已結束存檔。
- 同一玩家不可建立同名存檔。
### 3. 存檔隔離

每個存檔之資產、庫存、委託單、交易紀錄、自選股皆完全獨立，不得跨存檔存取。

### 4. 待成交委託清單

- 玩家下單後，系統將委託單列入「待成交委託」清單。
- 只要委託單狀態為 `PENDING`，且尚未進入該階段結算前，玩家即可撤單。
- 一旦系統開始進行該階段結算，該階段所有 `PENDING` 委託單即不可再撤。

### 5. 存檔列表資訊
(設計前端待討論)
存檔列表至少需揭露以下資訊：

- 存檔名稱
- 當前虛擬日期
- 當前交易階段(可能不用)
- 總資產價值
- 累積報酬率
- 存檔狀態

其中：

- 總資產價值 = 存款戶餘額 + 交割戶餘額 + 持股市值
- 累積報酬率 = (目前總資產 - 初始資金) / 初始資金

### 6. 存檔生命週期 (Status)

- `ACTIVE`：進行中，玩家可正常推進時間與交易。
- `BANKRUPT`：已破產。玩家無法再進行任何交易或推進時間，僅能查詢、刪除或保留封存。
- `FINISHED`：玩家手動結束並封存該存檔，不可再恢復進行。

### 7. 刪除存檔

- 玩家可主動刪除任一存檔。
- 刪除後該存檔資料自系統中移除，並釋放存檔名額。

## 三、遊戲核心與目標

### 1. 遊戲核心循環 (Core Loop)

登入系統 -> 選擇或建立存檔 -> 研究歷史盤面 -> 做出買賣決策 -> 點擊推進時光 -> 檢視撮合結果與損益 -> 重複循環。

研究歷史盤面，指查看股價和 K 線圖

### 2. 遊戲目標

本遊戲為沙盒模擬，無絕對勝利條件。主要目標為追求更高的累積報酬率，並在限制條件下進行長期存活與資產成長。

### 3. 破產定義
交割戶不夠錢支付即破產

## 四、虛擬時間與階段切換

### 1. 推進規則

- 玩家以「交易階段」為單位推進時間。
- 階段切換順序固定為：
  `盤前 -> 盤中 -> 盤後 -> 收市 -> 下一有效交易日盤前`
- 一旦推進時間，不可回到過去日期。

<!-- - 玩家僅能在「收市」階段選擇跳轉至 N 天後，系統將自動跳轉至 N 天後的「盤前」階段。
- 若跳轉區間中包含非交易日，系統自動跳至下一個有效交易日。
- 玩家僅能在「收市」階段執行跳轉 N 天後，且當下不得存在尚未結算的委託單。 -->


### 2. 交易階段定義

| 階段名稱 | 虛擬時間範圍 | 系統行為限制 |
| --- | --- | --- |
| 盤前 | 08:30 - 09:00 | 允許掛限價單與撤單，不進行成交。 |
| 盤中 | 09:00 - 13:30 | 允許掛限價單、市價單與撤單。 |
| 盤後 | 14:00 - 14:30 | 僅允許定價交易，成交價固定使用收盤價。 |
| 收市 | 14:30 以後 | 鎖定所有交易功能，僅供查詢報價、損益與推進時間。 |

### 3. 當日可見資訊

- 在任一時間點，玩家僅可查詢當前虛擬日期以前已揭露之資訊。
- 當前日期尚未經過之盤中、盤後或收市結果，不得提前顯示。

## 五、資金與帳戶架構

### 1. 雙帳戶機制

- 存款戶 (Savings Account)：存放玩家暫時不投入交易之資金。
- 交割戶 (Trading Account)：所有證券買賣之扣款與入帳皆透過此帳戶。
- 兩帳戶間可即時轉帳，無手續費。

### 2. 資產估值

- 總資產 = 存款戶餘額 + 交割戶餘額 + 持股市值。
- 持股市值以當前已知之最新價格估算。
- 若處於盤前，持股市值以上一有效交易日收盤價估算。
- 若處於盤中，持股市值以當日開盤價估算。
- 若處於盤後或收市，持股市值以當日收盤價估算。

### 3. 交易成本

- 買進手續費：成交金額 × 0.1425%
- 賣出手續費：成交金額 × 0.1425%
- 賣出證交稅：成交金額 × 0.3%
- 單筆手續費若不足 20 元，以 20 元計

### 4. 交割制度

- 本版本採 T+0 瞬間交割。意即交易委託一旦經系統撮合成交，便立即於當日完成款券劃撥，無須等待如真實市場之 T+2 天。
- 實際交割款項與持股增減之時間點，發生於**「階段推進時的撮合結算點」**，而非下單委託的瞬間。
- 買入成交時：系統立即增加持股，並自交割戶扣除成交金額與手續費。若交割戶扣款後餘額為負，則判定破產。
- 賣出成交時：系統立即減少持股，並將成交金額扣除手續費與證交稅之淨額入帳至交割戶。
- 在交易委託尚未撮合成交前（處於 `PENDING` 狀態），系統不預先圈存（扣除）買單資金，亦不提前更新持股。

### 5. 餘額原則

- 存款戶與交割戶在正常狀態下不得為負。
- 若因批量結算導致交割戶餘額不足以支付應付款項，則該存檔直接判定破產，存檔狀態變更為 `BANKRUPT`。

## 六、交易標的限制

### 1. 可交易標的

本系統以台股上市、上櫃與系統資料庫中已定義之標的為限。

### 2. 禁止交易情況

以下情況不得下單：

- 該股票於當前虛擬日期尚未上市
- 該股票於當前虛擬日期暫停交易
- 當前為非允許下單之交易階段

## 七、委託單與撮合邏輯

### 1. 支援委託類型

- 限價單 (Limit Order)：玩家指定價格
- 市價單 (Market Order)：玩家不指定價格，由系統依規則決定成交價
- 盤後定價交易：視為特殊形式之市價交易，成交價固定為收盤價

### 2. 委託單共通規則

- 本版本所有委託單皆為整股交易。
- 本版本不支援部分成交；委託單只有「全部成交」或「全部不成交」兩種結果。
- 委託單皆視為當日有效，當對應階段結束後若未成交，則自動失效。
- 玩家可於對應撮合前主動撤單。
- 買單允許下單時不先圈存現金。
- 賣單不允許超額下單。若玩家持有 1000 股，則不可送出 2000 股賣單。
- 送出賣單時，系統必須檢查當前可賣持股是否足夠；若不足，拒絕下單。
- 可賣持股數量 = 現有持股數量 - 同階段尚未結算之賣單總量。

### 3. 限價單價格限制

限價單價格須符合台股升降單位：

| 股價範圍 | 升降單位 |
| --- | --- |
| 未滿 10 元 | 0.01 元 |
| 10 元至未滿 50 元 | 0.05 元 |
| 50 元至未滿 100 元 | 0.1 元 |
| 100 元至未滿 500 元 | 0.5 元 |
| 500 元至未滿 1000 元 | 1 元 |
| 1000 元以上 | 5 元 |

其他限制：

- 指定價格不得高於當日漲停價。
- 指定價格不得低於當日跌停價。
- 系統可提供「漲停價」與「跌停價」快捷選項。

### 4. 各交易階段可下單類型

| 階段名稱 | 可下單類型 |
| --- | --- |
| 盤前 | 限價單 |
| 盤中 | 限價單、市價單 |
| 盤後 | 定價交易 |

### 5. 撮合邏輯

#### (1) 盤前限價買單

- 下單時機：玩家於盤前送出限價買單。
- 撮合時機：當玩家推進至盤中時進行結算。
- 成交條件：限價 >= 當日開盤價。
- 實際成交價：當日開盤價。
- 未成交條件：限價 < 當日開盤價。
- 成交後立即扣除成交金額與手續費，若交割戶不足則破產。

#### (2) 盤前限價賣單

- 下單時機：玩家於盤前送出限價賣單。
- 撮合時機：當玩家推進至盤中時進行結算。
- 成交條件：限價 <= 當日開盤價。
- 實際成交價：當日開盤價。
- 未成交條件：限價 > 當日開盤價。
- 成交後立即扣除賣出手續費與證交稅，餘額入帳至交割戶。

#### (3) 盤中限價買單

- 下單時機：玩家於盤中送出限價買單。
- 撮合時機：當玩家推進至盤後時進行結算。
- 成交條件：限價 >= 當日最低價。
- 實際成交價：玩家指定限價。
- 未成交條件：限價 < 當日最低價。
- 本系統不提供價格改善，以避免玩家藉由歷史最低價取得不合理優勢。
- 成交後立即扣除成交金額與手續費，若交割戶不足則破產。

#### (4) 盤中限價賣單

- 下單時機：玩家於盤中送出限價賣單。
- 撮合時機：當玩家推進至盤後時進行結算。
- 成交條件：限價 <= 當日最高價。
- 實際成交價：玩家指定限價。
- 未成交條件：限價 > 當日最高價。
- 成交後立即扣除賣出手續費與證交稅，餘額入帳至交割戶。

#### (5) 盤中市價買單

- 下單時機：玩家於盤中送出市價買單。
- 撮合時機：當玩家推進至盤後時進行結算。
- 成交條件：當日存在有效成交價格資料。
- 實際成交價：當日收盤價。
- 若當日無法交易，則該委託單失效。
- 成交後立即扣除成交金額與手續費，若交割戶不足則破產。

#### (6) 盤中市價賣單

- 下單時機：玩家於盤中送出市價賣單。
- 撮合時機：當玩家推進至盤後時進行結算。
- 成交條件：當日存在有效成交價格資料。
- 實際成交價：當日收盤價。
- 若當日無法交易，則該委託單失效。
- 成交後立即扣除賣出手續費與證交稅，餘額入帳至交割戶。

#### (7) 盤後定價買單

- 下單時機：玩家於盤後送出買單。
- 撮合時機：當玩家推進至收市時進行結算。
- 成交條件：當日存在有效收盤價資料。
- 實際成交價：當日收盤價。
- 若當日無法交易，則該委託單失效。
- 成交後立即扣除成交金額與手續費，若交割戶不足則破產。

#### (8) 盤後定價賣單

- 下單時機：玩家於盤後送出賣單。
- 撮合時機：當玩家推進至收市時進行結算。
- 成交條件：當日存在有效收盤價資料。
- 實際成交價：當日收盤價。
- 若當日無法交易，則該委託單失效。
- 成交後立即扣除賣出手續費與證交稅，餘額入帳至交割戶。

### 6. 委託單狀態

本版本委託單至少有以下狀態：

- `PENDING`：待撮合
- `FILLED`：已成交
- `CANCELED`：玩家主動撤單 
- `EXPIRED`：階段結束仍未成交，自動失效

## 八、自選股（觀察清單）管理

### 1. 基本功能

- 玩家可將股票加入自選股清單。
- 玩家可自自選股清單移除股票。
- 自選股清單隸屬於個別存檔，不跨存檔共享。

### 2. 限制

- 同一存檔內，同一檔股票不得重複加入自選股。
- 每個存檔的自選股最多 <font color="red">100 </font> 筆。

## 九、K 線圖與技術指標

### 1. K 線圖功能

- 玩家可查詢日線、週線、月線圖。
- K 線圖主圖顯示股價資訊，包含開盤價、最高價、最低價、收盤價，以及由此組成之 K 棒。
- K 線圖下方副圖顯示成交量（Volume），每一根成交量柱對應主圖同週期的一根 K 棒。
- 成交量的時間週期需與 K 線主圖一致：若主圖為日 K，則下方顯示日成交量；若主圖為週 K，則下方顯示週成交量；若主圖為月 K，則下方顯示月成交量。
- 玩家可疊加常見技術指標，如移動平均線、RSI、MACD。
- 技術指標計算僅能使用當前虛擬日期以前之歷史資料，不得使用未來資料。

### 2. 當日 K 線揭露原則
K線僅顯示前一交易日以前的資料

### 3. 成交量顯示原則

- 成交量定義為在特定時間週期內實際成交之數量。
- 本系統之成交量圖顯示於 K 線主圖下方，作為股價走勢之輔助觀察資訊。
- 成交量柱之長短代表該週期成交量相對大小。
- 成交量圖可作為玩家判讀市場活絡程度與流動性之參考，但不保證可單獨預測未來漲跌。

### 4. 參考來源

- 學吧 LearningPa，〈看完它，你將學會股票成交量的2種解析2個狀況1個應用〉：
  https://learningpa.cc/tradingvolume/
- 其中「1・成交量是什麼？」段落提到，成交量是成交的數量，且實際圖表中成交量通常顯示在 K 線圖下方。

## 十、MVP 實作原則

- 優先確保規則一致性，再追求真實市場細節。
- 優先完成可完整遊玩的單人模擬流程。
- 所有規則若與真實市場細節衝突，以本白皮書之 MVP 規則為準。
