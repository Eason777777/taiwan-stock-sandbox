SQL 查詢解析：自選股估價一次 JOIN

資料表關聯圖

watchlists w
  • save_id  ← WHERE 篩選
  • stock_id → 驅動 JOIN
INNER JOIN
stock_id
stocks s
  • stock_name_zh
  • market_type
  • sector_name
LEFT JOIN
stock_id + price_date
daily_prices dp_current
  • trade_date = :price_date
  • open/close → current_price
  • is_attention
  • is_disposition
  • is_full_delivery
LEFT JOIN
stock_id + today
daily_prices dp_today
  • trade_date = :current_trade_date
  • ref_price → 漲跌基準

為什麼這樣 JOIN？

watchlists（驅動表）

用 WHERE w.save_id = ? 篩出這個存檔的所有自選股。 每一列代表「這個玩家關注了哪支股票」。

INNER JOIN stocks

取得股票的靜態資料：中文名稱、市場別、產業別。 用 INNER JOIN 是因為每支自選股一定存在於 stocks 表，否則資料本身就壞了。

LEFT JOIN dp_current（估價基準日）

取得「估價基準日」當天的價格與警告旗標。
trade_date = :price_date — 這個日期因交易階段而異（見下表）。
用 LEFT JOIN 是因為若該股當天停牌，允許回傳 null 而非整列消失。

LEFT JOIN dp_today（今日）

單獨取今日的 ref_price（開盤參考價）。
即使在盤前（估價基準日 = 昨天），計算漲跌幅時仍需今日的參考價。 這就是為什麼要 JOIN 兩次 daily_prices。

交易階段與 :price_date 的對應

current_phase :price_date（估價基準日） price_col（取哪個價格） price_change 計算
PRE_MARKET 盤前 前一有效交易日
MAX(trade_date < today)		 close_price 昨日收盤 固定 0（尚未開盤）
INTRADAY 盤中 今日（= current_trade_date） open_price 今日開盤 open_price − ref_price
POST_MARKET / CLOSED 盤後 今日（= current_trade_date） close_price 今日收盤 close_price − ref_price
