📖專案簡介

錢錢錢市是一個零風險的股票模擬交易平台。玩家自由選擇歷史時間起點，在「平行宇宙」中逐階段推進虛擬時間，並以真實的台股歷史價格進行委託撮合。核心強調 時空隔離（無法利用未來資訊套利）與 邏輯防呆（在簡化的 MVP 模型下維持金融運作嚴謹性）。

🕰 時間模擬
自選歷史起始日，每次呼叫 /advance 推進一個交易階段（盤前→盤中→盤後→收市→下一日）
📊 階段化撮合
各階段委託於推進至下一階段時結算，限價、市價與盤後定價規則分明
💰 雙帳戶制
存款戶（閒置資金）與交割戶（買賣專用），即時轉帳、完全隔離
🗂 多存檔
每個存檔是完全獨立的模擬，資金、持股、委託、自選股互不影響
MVP 範圍：本版本不考慮除權息、減資、T+2 交割、融資融券、選擇權／期貨、圈存、IOC／FOK、零股與部分成交。規則與真實市場衝突時，以白皮書 MVP 規則為準。
⚠ 注意：資料庫位於 mysql.shiragaserver.lan，僅可透過 Tailscale 連線。本機開發前請確認 Tailscale 已啟動。

🔐認證 /auth

支援玩家自助註冊：輸入帳號（account）與密碼（password），系統檢查後即註冊成功。所有 API（除了 /auth/register 與 /auth/login）都需要在 Header 帶上 session token：

X-Session-Id: <your_session_id>

單一登入原則：同一帳號同一時間僅允許一組有效憑證。再次登入會覆蓋前一次憑證，舊 token 立即失效（避免異地登入與異地操作）。

方法路由說明需要 Session
POST /auth/register 建立帳號 否
POST /auth/login 登入，回傳 session_id 否
POST /auth/logout 登出，清除 session 是

範例

# 註冊
curl -X POST http://localhost:8000/auth/register \
  -H "Content-Type: application/json" \
  -d '{"account": "alice", "password": "mypassword"}'

# 登入
curl -X POST http://localhost:8000/auth/login \
  -H "Content-Type: application/json" \
  -d '{"account": "alice", "password": "mypassword"}'
# → {"session_id": "abc123..."}

# 登出
curl -X POST http://localhost:8000/auth/logout \
  -H "X-Session-Id: $SESSION"

測試指令

# 1. 註冊
curl -s -X POST http://localhost:8000/auth/register \
  -H "Content-Type: application/json" \
  -d '{"account": "testuser", "password": "mypassword"}'

# 2. 重複註冊（預期 409）
curl -s -X POST http://localhost:8000/auth/register \
  -H "Content-Type: application/json" \
  -d '{"account": "testuser", "password": "mypassword"}'

# 3. 登入
curl -s -X POST http://localhost:8000/auth/login \
  -H "Content-Type: application/json" \
  -d '{"account": "testuser", "password": "mypassword"}'

# 4. 登入後登出（將上一步拿到的 session_id 帶入）
SESSION_ID="<上一步拿到的 session_id>"
curl -s -X POST http://localhost:8000/auth/logout \
  -H "X-Session-Id: $SESSION_ID"

一行完成完整流程

curl -s -X POST http://localhost:8000/auth/register \
  -H "Content-Type: application/json" \
  -d '{"account": "testuser", "password": "mypassword"}'

SESSION_ID=$(curl -s -X POST http://localhost:8000/auth/login \
  -H "Content-Type: application/json" \

  | python3 -c "import sys,json; print(json.load(sys.stdin)['session_id'])")

💾模擬存檔 /saves

每個存檔是一次完全獨立的模擬，其資產、庫存、委託單、交易紀錄、自選股皆不得跨存檔存取。

方法路由說明
POST/saves建立新存檔
GET/saves列出所有存檔
GET/saves/{id}查看單一存檔
POST/saves/{id}/advance推進至下一交易階段
DELETE/saves/{id}刪除存檔（釋放名額）
建檔名額
最多 5 個進行中存檔；最多保留 10 個（含進行中、破產、結束）；同玩家不可同名。
初始日期
須介於資料庫最早與最後可用交易日之間，且為有效交易日；可由系統隨機抽取。
起始資金
50,000 ~ 1,000,000 元，可由系統隨機決定；全數配發至存款戶。
列表揭露
名稱、虛擬日期、總資產、累積報酬率、狀態。報酬率 =（總資產 − 初始資金）/ 初始資金。

存檔生命週期（status）

  • ACTIVE — 進行中，可正常推進時間與交易。
  • BANKRUPT — 已破產。無法再交易或推進時間，僅能查詢、刪除或封存。
  • FINISHED — 玩家手動結束並封存，不可再恢復進行。

交易階段循環

盤前
限價單
→
盤中
限價＋市價
→
盤後
定價交易
→
收市
鎖定交易
→
下一日盤前
重複

每次推進都會結算「上一階段送出且尚未結算的委託單」，未成交者自動失效（詳見撮合規則）。一旦推進時間，不可回到過去日期。

# initial_funds 範圍 50,000 ~ 1,000,000，全數配發至存款戶

💳帳戶與資金 /saves/{id}/accounts

每個存檔有兩個帳戶，兩者間可即時轉帳、無手續費。本版本採 T+0 瞬間交割。

存款戶（savings）
閒置資金存放處。無法直接用於股票買賣。
交割戶（trading）
股票買賣、手續費、交易稅專用帳戶。餘額不足以支付即破產。
資產估值：持股市值依當前階段採不同價格 —— 盤前以上一交易日收盤價、盤中以當日開盤價、盤後／收市以當日收盤價估算。總資產 = 存款戶 + 交割戶 + 持股市值。
方法路由說明
POST/saves/{id}/accounts/transfer兩帳戶間轉帳
GET/saves/{id}/accounts/history查看帳戶異動紀錄

轉帳

# 存款戶 → 交割戶
curl -X POST http://localhost:8000/saves/1/accounts/transfer \
  -H "Content-Type: application/json" \
  -H "X-Session-Id: $SESSION" \
  -d '{"direction": "savings_to_trading", "amount": 500000}'

# 交割戶 → 存款戶
curl -X POST http://localhost:8000/saves/1/accounts/transfer \
  -H "Content-Type: application/json" \
  -H "X-Session-Id: $SESSION" \
  -d '{"direction": "trading_to_savings", "amount": 100000}'

📈股票查詢 /stocks

方法路由說明
GET/stocks?q=&sector=搜尋股票（代號或名稱 / 類股）
GET/stocks/{stock_id}查看股票基本資料

📋委託下單 /saves/{id}/orders

下單限制：所有委託為當日有效（ROD）、整股交易，且只有「全部成交」或「全部不成交」（不支援部分成交）。委託可於對應撮合前主動撤單。可交易標的須已上市、未暫停交易，且當前為允許下單之階段。
方法路由說明
POST/saves/{id}/orders送出委託（限價 / 市價 / 定價）
GET/saves/{id}/orders查看所有委託
DELETE/saves/{id}/orders/{oid}撤銷 PENDING 委託（該階段結算前）

各階段可下單類型

階段可下單類型
盤前限價單
盤中限價單、市價單
盤後定價交易（成交價固定為收盤價）

限價單升降單位

限價單價格須符合台股升降單位，且不得高於當日漲停價、不得低於當日跌停價。

股價範圍升降單位
未滿 10 元0.01 元
10 ~ 未滿 50 元0.05 元
50 ~ 未滿 100 元0.1 元
100 ~ 未滿 500 元0.5 元
500 ~ 未滿 1000 元1 元
1000 元以上5 元
賣單檢查：買單下單時不先圈存現金；賣單不允許超額。送出賣單時系統檢查可賣持股是否足夠，不足即拒絕。
可賣持股 = 現有持股 − 同階段尚未結算之賣單總量

限價買單

curl -X POST http://localhost:8000/saves/1/orders \
  -H "Content-Type: application/json" \
  -H "X-Session-Id: $SESSION" \
  -d '{
    "stock_id":   "2330",
    "order_type": "LIMIT",
    "side":       "BUY",
    "price":      330.0,
    "quantity":   1
  }'
# quantity 單位為張（1 張 = 1,000 股）

市價賣單

curl -X POST http://localhost:8000/saves/1/orders \
  -H "Content-Type: application/json" \
  -H "X-Session-Id: $SESSION" \
  -d '{
    "stock_id":   "2330",
    "order_type": "MARKET",
    "side":       "SELL",
    "quantity":   1
  }'
# 市價單不填 price；盤中市價以當日收盤價成交

撮合規則（階段化結算）

委託送出時列入「待成交委託」（PENDING），於玩家推進至下一階段時結算：

委託撮合時機成交條件成交價
盤前 限價買推進至盤中限價 ≥ 開盤價當日開盤價
盤前 限價賣推進至盤中限價 ≤ 開盤價當日開盤價
盤中 限價買推進至盤後限價 ≥ 最低價玩家指定限價
盤中 限價賣推進至盤後限價 ≤ 最高價玩家指定限價
盤中 市價買/賣推進至盤後當日有有效價格當日收盤價
盤後 定價買/賣推進至收市當日有有效收盤價當日收盤價
  • 盤中限價單不提供價格改善（避免藉歷史最低價套利）。
  • 市價／定價單若當日無法交易則委託失效。
  • 買單成交後立即扣成交金額與手續費；交割戶不足則破產。
  • 賣單成交後立即扣手續費與證交稅，餘額入帳交割戶。

委託單狀態

  • PENDING — 待撮合
  • FILLED — 已成交
  • CANCELED — 玩家主動撤單
  • EXPIRED — 階段結束仍未成交，自動失效

⚖業務規則

  • 交易單位 1 張 = 1,000 股，quantity 以張為單位，僅整股交易，不支援零股。
  • ROD 委託：所有委託當日有效，對應階段結束仍未成交即自動失效（EXPIRED）。
  • All-or-Nothing：委託全量成交或全量不成交，不支援部分成交（StockOrder → StockTransaction 為 1:1）。
  • 升降單位＋漲跌停：限價單須符合台股升降單位，且不得高於漲停價、不得低於跌停價，超出即拒絕。
  • 賣單不可超額：可賣持股 = 現有持股 − 同階段未結算賣單量；不足則拒絕。買單下單時不圈存現金。
  • T+0 瞬間交割：成交立即更新持股與交割戶。帳戶正常狀態不得為負，交割戶不足以支付即破產。
  • 手續費：買賣各收 0.1425%（最低 NT$20）；交易稅：賣出 0.3%（僅賣方）。金額存為整數（元）。
  • 交易標的限制：尚未上市、暫停交易，或非允許下單階段，皆不得下單。
  • 時空隔離：僅可查詢當前虛擬日期以前已揭露之資訊；未來盤中／盤後／收市結果不得提前顯示。
  • 存檔隔離：各存檔資產、庫存、委託、交易紀錄、自選股完全獨立，不得跨存檔存取。

🔄典型使用流程

  1. 呼叫 POST /auth/register 建立帳號，POST /auth/login 取得 session_id
  2. 呼叫 POST /saves 建立存檔，指定起始日期與初始資金
  3. 呼叫 POST /saves/{id}/accounts/transfer，將資金從存款戶移入交割戶
  4. 呼叫 GET /stocks?q=... 或 /stocks/{id}/prices 研究歷史盤面與 K 線
  5. 於各階段呼叫 POST /saves/{id}/orders 掛委託（盤前限價、盤中限價／市價、盤後定價）
  6. 呼叫 POST /saves/{id}/advance 推進階段，系統結算上一階段委託
  7. 檢視撮合結果與損益（GET /saves/{id}/orders、/holdings）
  8. 持續推進 盤前 → 盤中 → 盤後 → 收市 → 下一交易日盤前
  9. 呼叫 GET /saves/{id}/holdings 查看持股與未實現損益
  10. 重複循環，追求更高累積報酬率並避免破產（交割戶不足即 BANKRUPT）
