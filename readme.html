📖專案簡介

錢錢錢市是一個零風險的股票模擬交易平台。使用者選擇一個歷史起始日期後，可逐日手動推進虛擬時間，並以真實的台股 OHLC 歷史資料進行委託撮合。

🕰 時間模擬
自選歷史起始日，每次呼叫 /advance 推進一個交易階段（盤前→盤中→盤後→下一日）
📊 真實資料
撮合以當日 DailyPrice 的最高/最低價為依據，限價與市價委託規則分明
💰 雙帳戶制
存款戶（閒置資金）與交割戶（買賣專用），手動轉帳、完全隔離
🗂 多存檔
每個存檔是完全獨立的模擬，資金、持股、委託互不影響
⚠ 注意：資料庫位於 mysql.shiragaserver.lan，僅可透過 Tailscale 連線。本機開發前請確認 Tailscale 已啟動。

🚀快速啟動

後端（FastAPI）

# 安裝依賴
pip install -r requirements.txt

# 選填：覆蓋 SQL API 位址（預設已指向 sql-api.shiragaserver.lan）
export SQL_API_URL="http://sql-api.shiragaserver.lan/query"

# 啟動開發伺服器
uvicorn app.main:app --reload

後端跑在 http://localhost:8000，Swagger UI 在 http://localhost:8000/docs。

前端（Vue 3 + Vite）

cd frontend
pnpm install
pnpm dev      # dev server → http://localhost:5173

  -H "Content-Type: application/json" \

🔐認證 /auth

所有 API（除了 /auth/register 與 /auth/login）都需要在 Header 帶上 session token：

X-Session-Id: <your_session_id>

每次登入都會輪換新 token，舊 token 立即失效（強制單一登入）。

方法路由說明需要 Session
POST /auth/register 建立帳號 否
POST /auth/login 登入，回傳 session_id 否
POST /auth/logout 登出，清除 session 是

範例

# 註冊
curl -X POST http://localhost:8000/auth/register \
  -H "Content-Type: application/json" \
  -d '{"account": "alice", "password": "mypassword"}'

# 登入
curl -X POST http://localhost:8000/auth/login \
  -H "Content-Type: application/json" \
  -d '{"account": "alice", "password": "mypassword"}'
# → {"session_id": "abc123..."}

# 登出
curl -X POST http://localhost:8000/auth/logout \
  -H "X-Session-Id: $SESSION"

💾模擬存檔 /saves

每個存檔是一次完全獨立的模擬，擁有各自的時間軸、資金與持股。

方法路由說明
POST/saves建立新存檔
GET/saves列出所有存檔
GET/saves/{id}查看單一存檔
POST/saves/{id}/advance推進至下一交易階段
DELETE/saves/{id}刪除存檔

交易階段循環

盤前
可下單
→
盤中
可下單
→
盤後
自動撮合
→
下一日盤前
重複

盤後結束時，系統自動以當日歷史價格撮合委託，未成交者失效。

建立存檔

curl -X POST http://localhost:8000/saves \
  -H "Content-Type: application/json" \
  -H "X-Session-Id: $SESSION" \
  -d '{
    "save_name": "我的第一次模擬",
    "start_date": "2020-01-02",
    "initial_savings": 1000000,
    "initial_trading": 0
  }'

查看存檔回應

{
  "save_id":        1,
  "save_name":      "我的第一次模擬",
  "current_date":   "2020-01-02",
  "current_phase":  "盤前",
  "status":         "進行中",
  "savings_balance": 1000000,
  "trading_balance": 0
}

💳帳戶與資金 /saves/{id}/accounts

每個存檔有兩個帳戶：

存款戶（savings）
閒置資金存放處。無法直接用於股票買賣。
交割戶（trading）
股票買賣、手續費、交易稅專用帳戶。餘額不足時買單被拒。
方法路由說明
POST/saves/{id}/accounts/transfer兩帳戶間轉帳
GET/saves/{id}/accounts/history查看帳戶異動紀錄

轉帳

# 存款戶 → 交割戶
curl -X POST http://localhost:8000/saves/1/accounts/transfer \
  -H "Content-Type: application/json" \
  -H "X-Session-Id: $SESSION" \
  -d '{"direction": "savings_to_trading", "amount": 500000}'

# 交割戶 → 存款戶
curl -X POST http://localhost:8000/saves/1/accounts/transfer \
  -H "Content-Type: application/json" \
  -H "X-Session-Id: $SESSION" \
  -d '{"direction": "trading_to_savings", "amount": 100000}'

📋委託下單 /saves/{id}/orders

下單限制：僅在盤前或盤中階段有效。所有委託為當日有效（ROD），盤後自動失效。委託價格須在漲跌停範圍內。
方法路由說明
POST/saves/{id}/orders送出委託（限價 / 市價）
GET/saves/{id}/orders查看所有委託
DELETE/saves/{id}/orders/{oid}撤銷待成交委託

限價買單

curl -X POST http://localhost:8000/saves/1/orders \
  -H "Content-Type: application/json" \
  -H "X-Session-Id: $SESSION" \
  -d '{
    "stock_id":   "2330",
    "order_type": "限價",
    "side":       "買",
    "price":      330.0,
    "quantity":   1
  }'
# quantity 單位為張（1 張 = 1,000 股）

市價賣單

curl -X POST http://localhost:8000/saves/1/orders \
  -H "Content-Type: application/json" \
  -H "X-Session-Id: $SESSION" \
  -d '{
    "stock_id":   "2330",
    "order_type": "市價",
    "side":       "賣",
    "quantity":   1
  }'
# 市價單不填 price，以當日最高/最低之間的隨機價格成交

撮合規則

  • 限價買單：委託價 ≥ 當日最低價 → 成交（成交價 = 委託價）
  • 限價賣單：委託價 ≤ 當日最高價 → 成交（成交價 = 委託價）
  • 市價單：成交價 = random(當日最低, 當日最高)
  • 委託為全量成交或全量不成交（All-or-Nothing）

🗺資料模型

共八個實體，SaveFile 為核心狀態機，所有模擬資源以 save_id 隔離。

User
user_id PK
account（unique）
password（hashed）
session_id（登入輪換）
Stock
stock_id PK（VARCHAR, e.g. 2330）
names, market_type, sector_name
listing_date, par_value
唯讀字典表
DailyPrice
(stock_id, trade_date) PK
open, high, low, close, volume
ref_price, limit_up, limit_down
撮合唯一依據
SaveFile
save_id PK
user_id FK
start_date, current_date, current_phase
savings_balance, trading_balance
StockOrder
order_id PK
save_id FK, stock_id FK
order_type（限價/市價）, side（買/賣）
price（nullable）, quantity, status
StockTransaction
transaction_id PK
order_id FK（1:1）
exec_price, quantity, fee, tax
avg_cost_at_transact
AccountTransaction
(save_id, seq) PK
每筆現金異動的帳本紀錄
含 balance_after 快照
Holding / Watchlist
(save_id, stock_id) PK
Holding: quantity, avg_cost
Watchlist: 純 M:N 關聯

⚖業務規則

  • 交易單位 1 張 = 1,000 股，quantity 以張為單位，不支援零股。
  • ROD 委託：所有委託當日有效，盤後若未成交自動失效。
  • All-or-Nothing：委託全量成交或全量不成交，不支援部分成交（StockOrder → StockTransaction 為 1:1）。
  • 價格檢查：委託價必須在 limit_up/limit_down 範圍內，超出即拒絕。
  • T+0 扣款：買單即時從交割戶扣款，餘額不足直接拒絕，帳戶餘額不可為負。
  • 手續費：買賣各收 0.1425%（最低 NT$20）；交易稅：賣出 0.3%（僅賣方）。金額存為整數（元）。
  • 上市日限制：委託日早於 listing_date 的股票直接拒絕。
  • 匹配原子性：盤後撮合在單一 DB transaction 中執行，不重複撮合，存檔完全隔離。
  • 模擬結束條件：到達資料集最晚日期，或交割戶 + 持股市值趨近 0。

🔄典型使用流程

  1. 呼叫 POST /auth/register 建立帳號，POST /auth/login 取得 session_id
  2. 呼叫 POST /saves 建立存檔，指定起始日期與初始資金
  3. 呼叫 POST /saves/{id}/accounts/transfer，將資金從存款戶移入交割戶
  4. 呼叫 GET /stocks?q=... 或 /stocks/{id}/prices 研究選股
  5. （盤前/盤中）呼叫 POST /saves/{id}/orders 掛委託
  6. 呼叫 POST /saves/{id}/advance 推進階段（盤前 → 盤中 → 盤後）
  7. 盤後自動撮合，系統依當日歷史價格決定成交與否
  8. 再次呼叫 POST /saves/{id}/advance，進入下一個交易日的盤前
  9. 呼叫 GET /saves/{id}/holdings 查看持股與未實現損益
  10. 重複步驟 4–9 直到模擬結束
小提示：Swagger UI（http://localhost:8000/docs）可以直接在瀏覽器試打所有 API，方便快速驗證功能。
